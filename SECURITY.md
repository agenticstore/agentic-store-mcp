# Security Policy

## Supported Versions

We currently support the latest major release of the application with security updates.

## Reporting a Vulnerability

If you've found a vulnerability or a potential vulnerability in AgenticStore MCP, please let us know immediately. We take security very seriously. 

Please do **NOT** report security vulnerabilities through public GitHub issues.

Instead, please report them to us by creating a private security advisory through the GitHub repository's "Security" tab, or by contacting the maintainers directly.

Include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We will prioritize the issue and get back to you as soon as possible, confirming receipt of your report and providing an estimated timeframe for a fix. After a fix is applied and tested, we will release a new version and, in conjunction with the reporter, publicly disclose the vulnerability if appropriate.
