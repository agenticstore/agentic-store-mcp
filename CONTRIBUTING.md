# Contributing to AgenticStore MCP

First off, thank you for considering contributing to AgenticStore MCP. It's people like you that make this tool great!

## Getting Started

1. Fork the repository on GitHub.
2. Clone the forked repository to your local machine.
3. Install the dependencies using `uv` or `pip` (e.g., `uv sync`).
4. Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name`.

## Making Changes

* Write tests for any new features or bug fixes.
* Ensure all tests pass before submitting your pull request.
* Follow the existing code style. We use standard formatting tools (e.g., `ruff`).
* Document any new features or changes in configuration in `README.md` or the relevant files.

## Submitting a Pull Request

1. Push your changes to your fork on GitHub.
2. Open a Pull Request against the `main` branch of the AgenticStore MCP repository.
3. Provide a clear and detailed description of your changes in the Pull Request description (use the provided PR template). Let us know what issue you're fixing, if applicable.
4. The core team will review your PR and might request changes before merging.

## Reporting Bugs

* Make sure you're testing against the latest version.
* Use the GitHub Issue Tracker to report bugs (use the Bug report template).
* Describe the bug clearly, including steps to reproduce, expected behavior, and actual behavior.
* Run relevant commands or include log snippets to help debug.

## Requesting Features

* Use the GitHub Issue Tracker to request new features (use the Feature request template).
* Provide a clear and detailed description of the feature requested, and why it would be useful to the community.
